Servlet - Java 페이지에 html 이 있는 것
JSP - html 페이지에 Java 코드가 있는 것


Get 방식
클라이언트에서 서버로 데이터를 전송할 때 URL에 파라미터를 함께 전송
주소창에 그대로 보이기 때문에 보안성이 떨어진다
길이에 제한이 있어서 전송 데이터의 한계가 있다
Post 방식보다 상대적으로 전송 속도가 빠르다

Post 방식
일정 크기 이상의 데이터를 보내야할 때 사용
서버로 보내기 전에 인코딩하고 전송 후 서버에서 디코딩 작업을 한다
Get 방식보다 느리다

Get 방식은 주로 웹 서버에 데이터를 요청할 때 사용
Post 방식은 웹 서버에 데이터를 전달하기 위해 사용


Session 
- 특정 웹사이트에서 사용자가 머무르는 기간 
- Session에 관련된 데이터는 서버에 저장
- 웹 브라우저의 캐시에 저장되어 브라우저가 닫히거나 서버에서 삭제시 사라진다
- Cookie에 비해 보안성이 좋다

Cookie 
- 인터넷 사용자가 특정 서버에 접속할 때 생성되는 개인 아이디와 비밀번호, 방문한 사이트의 정보를 담은 임시파일
- 서버가 아닌 클라이언트에 저장

모든 정보를 세션에 저장하면 서버의 메모리를 과도하게 사용 -> 서버 과부하


MVC 패턴
- 효율적인 개발과 유지보수를 위해서 사용하는 소프트웨어 디자인 패턴
Model - View - Controller 로 구성
